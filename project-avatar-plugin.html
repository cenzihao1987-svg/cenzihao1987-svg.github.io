中东版权头像协作工具设计

Vibe Coding × Figma Plugin

项目概述

这是一个围绕中东版权头像素材协作场景搭建的 Figma 插件。目标不是重做素材管理系统，而是在不改变运营、法务和设计师原有工作方式的前提下，把“可用头像”直接带进设计流程。

运营和法务继续在飞书多维表格维护授权状态，设计师则在 Figma 内完成筛选、预览、插入和信息生成，减少跨工具来回查找和反复确认。

01｜项目背景

受沙特当地文化与版权环境影响，女性人物头像获取难、授权周期短，部分素材来自采购、网红合作或阶段性授权，需要持续更新使用状态。

在地运营通过飞书多维表格管理头像来源、授权期限和上下架状态，但 UI 设计师与运营设计师在实际使用时，需要频繁往返于 Figma 和表格之间查找、下载和核对素材。部分已失效头像还可能遗留在历史设计稿中，带来版权和法务风险。

问题：如何在不改变大家的工作习惯前提下，提高协作效率？

02｜核心问题

素材分散

素材沉在大型表格中，查找和下载效率低，设计师很难快速找到合适头像。

状态滞后

授权状态变化后，设计师无法在设计现场即时获知，容易继续使用已失效素材。

人工核对重

设计完成后仍需与运营反复对表确认，协作成本高且容易遗漏。

本地化信息缺失

在地姓名、人物身份等信息缺少统一快捷的生成方式，影响设计产出效率。

03｜解决方案

通过 Vibe Coding 从 0 到 1 搭建 Figma 插件，并连接现有飞书多维表格。

运营和法务继续在原有表格中维护素材及版权状态，插件仅同步当前可用头像。设计师无需离开 Figma，即可完成头像筛选、预览和插入，同时生成符合当地习惯的随机姓名及身份信息。

04｜核心功能

  • 按国家、性别和人物类型筛选头像；
  • 实时同步素材授权及上下架状态；
  • 一键将头像和人物信息插入 Figma；
  • 生成符合中东地区习惯的姓名与基础资料；
  • 在插件内展示素材可用状态，减少人工确认。

05｜设计价值

该方案没有改变各角色原有的工作方式：运营和法务继续维护表格，设计师继续在 Figma 中完成设计。

通过打通素材管理与设计使用流程，项目减少了跨工具查找、重复核对和沟通返工，提高了三方协作效率，也降低了过期或下架素材被误用的法务风险。

对设计师在 Figma 内直接完成筛选、预览、插入与信息生成
对运营继续沿用原有飞书表格，不需要迁移协作流程
对法务把授权状态前置到使用现场，降低误用与遗留风险
对团队减少跨工具往返和重复确认，提升协作闭环效率

06｜项目总结

通过 Vibe Coding 将现有版权头像库转化为可直接在 Figma 中使用的插件，在保留原有工作习惯的基础上，实现素材状态实时同步、快速调用与合规使用。

这个项目的价值不在于额外造了一个管理平台，而在于把已经存在的协作规则前移到了设计动作发生的地方。
